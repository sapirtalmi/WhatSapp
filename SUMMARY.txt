AUTH FLOW
---------
Register -> POST /auth/register -> auto-login -> POST /auth/login -> JWT stored in
localStorage -> axios interceptor attaches Bearer token to every request ->
GET /users/me validates token and hydrates user state on page load.

OWNERSHIP RULES
---------------
- PATCH / DELETE: only owner (403 otherwise)
- GET single: owner always; non-owner only if is_public=true (403 otherwise)
- GET list: returns owned collections + all public collections from any user

STEP 4 — Places + PostGIS
========================================

BACKEND
-------
app/schemas/place.py        - PlaceCreate (lat/lng floats + range validators), PlaceUpdate, PlaceOut
app/routers/places.py       - POST   /collections/{id}/places    — create place, stores SRID=4326 POINT
                              GET    /collections/{id}/places    — list; 403 if private & not owner
                              PATCH  /collections/{id}/places/{pid} — update; 403 if not owner
                              DELETE /collections/{id}/places/{pid} — delete; 403 if not owner (204)
                              GET    /places/nearby?lat=&lng=&radius= — ST_DWithin geography query,
                                     public collections only, radius in metres (max 50000)
main.py                     - places router included

GEO NOTES
---------
- Stored as: SRID=4326;POINT(lng lat)  ← note: PostGIS is (lng, lat) order
- Returned as: lat/lng floats via geoalchemy2 to_shape() → point.y / point.x
- Nearby query uses ST_DWithin with use_spheroid=True for accurate metre distances
- Migration needed: alembic revision --autogenerate -m "add places table"
  (adds GiST index on place.location automatically via GeoAlchemy2)

FRONTEND

STEP 6 — Friendship System
========================================

BACKEND
-------
app/schemas/friendship.py   - FriendshipOut, FriendUserOut, FriendshipWithUserOut
app/routers/friends.py      - POST   /friends/request/{id}  — send request (409 if exists, 400 if self)
                              PUT    /friends/{id}/accept   — addressee only; pending → accepted
                              PUT    /friends/{id}/reject   — addressee only; deletes record
                              PUT    /friends/{id}/block    — either party; sets blocked status
                              DELETE /friends/{id}          — unfriend; either party (204)
                              GET    /friends               — accepted friends with other user info
                              GET    /friends/pending       — incoming pending requests
app/routers/users.py        - GET /users/search?q=         — case-insensitive username search,
                                                             excludes self, limit 20
main.py                     - friends router included

FRONTEND
--------
src/api/friends.js          - getFriends, getPendingRequests, sendFriendRequest,
                              acceptFriendRequest, rejectFriendRequest, removeFriend, searchUsers
src/pages/Friends.jsx       - search with debounce (400ms), pending requests inbox
                              (accept/decline), friends list with remove
                              all with optimistic UI updates
src/App.jsx                 - /friends route added, "Friends" button on Home

SOCIAL RULES
------------
- Only addressee can accept/reject a request
- Either party can block or unfriend
- Blocked status prevents new requests (403)
- Self-requests return 400
- Duplicate requests return 409
